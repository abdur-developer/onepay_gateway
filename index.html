Introduction

Our Payment Gateway allows you to accept secure online payments with simple API requests. It supports both server-side and client-side integrations with minimal setup required.

Note: Always verify payments from your own server. Never trust success_url alone — check the transaction on your server with the transaction ID (trx_id).

Installation

PHP

Make sure your PHP installation has cURL enabled:

sudo apt install php-curl

Test if cURL is enabled:

<?php phpinfo(); ?>
Python

Install the required package:

pip install requests
JavaScript

Make sure you are using a browser that supports Fetch API. All modern browsers support this.

For Node.js backend:

npm install axios

Required Parameters

Parameter Type Description
api_key string Your secure API key
amount float Payment amount (e.g., 40.00)
success_url string Redirect URL after successful payment
cancel_url string Redirect URL if payment is cancelled
my_data string Custom reference or tracking string

API Endpoint

POST https://onepay.website/api/create.php

API Response

Successful response:

{
  "success": true,
  "trx_id": "TX-UNIQUEID",
  "payment_url": "https://.../payment.php?trx=TX-UNIQUEID"
}

Error response:

{
  "success": false,
  "message": "Invalid API Key",
  "payment_url": "https://your-cancel-url.com"
}

Integration

PHP Server-Side Integration 

<?php
$api_key = 'YOUR_API_KEY';
$url = 'https://onepay.website/api/create.php';

$data = [
    'api_key'     => $api_key,
    'success_url' => 'https://yourdomain.com/success.php',
    'cancel_url'  => 'https://yourdomain.com/cancel.php',
    'my_data'     => 'my_custom_data',
    'type'        => 'sandbox', // Optional, use 'sandbox' for testing... 'type' can be 'sandbox' or 'live'
    'amount'      => 40
];

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, http_build_query($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/x-www-form-urlencoded',
    'User-Agent: Mozilla/5.0'
]);
$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

if ($http_code == 200) {
    $result = json_decode($response, true);
    header("Location: " . $result['payment_url']);
    exit;
} else {
    echo "HTTP Error $http_code";
}
?>

Python Server-Side Integration 

import requests

url = 'https://onepay.website/api/create.php'
data = {
    'api_key': 'YOUR_API_KEY',
    'success_url': 'https://yourdomain.com/success.php',
    'cancel_url': 'https://yourdomain.com/cancel.php',
    'my_data': 'my_custom_data',
    'type': 'sandbox',  # Optional, use 'sandbox' for testing... 'type' can be 'sandbox' or 'live'
    'amount': 40
}

response = requests.post(url, data=data, headers={
    'Content-Type': 'application/x-www-form-urlencoded',
    'User-Agent': 'Mozilla/5.0'
})

print(response.status_code)
print(response.json())

JavaScript Client-Side Integration 

const data = new URLSearchParams({
  api_key: 'YOUR_API_KEY',
  success_url: 'https://yourdomain.com/success.php',
  cancel_url: 'https://yourdomain.com/cancel.php',
  my_data: 'my_custom_data',
  type: 'sandbox', // Optional, use 'sandbox' for testing... 'type' can be 'sandbox' or 'live'
  amount: 40
});

fetch('https://onepay.website/api/create.php', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  },
  body: data
})
.then(res => res.json())
.then(res => {
  if (res.success) {
    window.location.href = res.payment_url;
  } else {
    alert('Payment failed: ' + res.message);
  }
})
.catch(err => alert('Request failed: ' + err));

Mobile App Integration

You can integrate this gateway into your Android or iOS app using standard HTTP clients.

Android (Kotlin) 
val client = OkHttpClient()

val formBody = FormBody.Builder()
    .add("api_key", "YOUR_API_KEY")
    .add("amount", "40")
    .add("success_url", "https://yourdomain.com/success")
    .add("cancel_url", "https://yourdomain.com/cancel")
    .add("my_data", "my_custom_data")
    .add("type", "sandbox") // Optional, use 'sandbox' for testing... 'type' can be 'sandbox' or 'live'
    .build()

val request = Request.Builder()
    .url("https://onepay.website/api/create.php")
    .post(formBody)
    .addHeader("Content-Type", "application/x-www-form-urlencoded")
    .build()

client.newCall(request).enqueue(object : Callback {
    override fun onFailure(call: Call, e: IOException) {
        // Handle error
    }
    
    override fun onResponse(call: Call, response: Response) {
        val json = JSONObject(response.body()?.string())
        if (json.getBoolean("success")) {
            val paymentUrl = json.getString("payment_url")
            // Open in browser or WebView
        }
    }
})
iOS (Swift) 
let url = URL(string: "https://onepay.website/api/create.php")!
var request = URLRequest(url: url)
request.httpMethod = "POST"
request.setValue("application/x-www-form-urlencoded", forHTTPHeaderField: "Content-Type")

let parameters = [
    "api_key": "YOUR_API_KEY",
    "amount": "40",
    "success_url": "https://yourdomain.com/success",
    "cancel_url": "https://yourdomain.com/cancel",
    "my_data": "my_custom_data",
    "type": "sandbox" // Optional, use 'sandbox' for testing... 'type' can be 'sandbox' or 'live'
]
request.httpBody = parameters
    .map { "\($0.key)=\($0.value)" }
    .joined(separator: "&")
    .data(using: .utf8)

let task = URLSession.shared.dataTask(with: request) { data, response, error in
    guard let data = data else { return }
    do {
        if let json = try JSONSerialization.jsonObject(with: data) as? [String: Any],
           let success = json["success"] as? Bool,
           success,
           let paymentUrl = json["payment_url"] as? String {
            DispatchQueue.main.async {
                UIApplication.shared.open(URL(string: paymentUrl)!)
            }
        }
    } catch {
        print("JSON error: \(error.localizedDescription)")
    }
}
task.resume()
